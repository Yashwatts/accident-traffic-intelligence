# Documentation

Comprehensive documentation for the Traffic & Accident Intelligence Platform.

## Files

- **architecture.md** - System architecture and design decisions
- **design-system.md** - UI design system (colors, typography, components)
- **responsive-layout-strategy.md** - Responsive design patterns
- **folder-structure.md** - Project folder organization
- **api.md** - API documentation (endpoints, requests, responses)
- **deployment.md** - Deployment guide (AWS, Docker, CI/CD)
- **contributing.md** - Contribution guidelines for developers

## Additional Documentation

- API documentation is auto-generated from code using Swagger/OpenAPI
- Component documentation is generated using Storybook
- Database schema is documented in the architecture file

## For Developers

1. Read **architecture.md** first to understand the system
2. Check **folder-structure.md** to navigate the codebase
3. Review **design-system.md** for UI guidelines
4. Follow **contributing.md** for pull request guidelines

## For Designers

1. **design-system.md** - Colors, typography, spacing
2. **responsive-layout-strategy.md** - Responsive patterns
3. Component examples in Storybook (run `npm run storybook`)

## For DevOps

1. **deployment.md** - Deployment instructions
2. **architecture.md** - Infrastructure requirements
3. Docker files in root directory
